04Motion

One signature easing — cubic-bezier(0.22, 1, 0.36, 1) — on everything that moves. Celebrations are one-shot, signal-keyed, and suppressed entirely under prefers-reduced-motion (the fallback is deliberately nothing; content still announces via aria-live). Any looping animation must be compositor-only (transform/opacity) — plan 018's rule after the 60fps background-position repaint was killed.

05Laws

The rules that keep the system coherent. Breaking one is a decision, not an accident — bring it up first.

  1. The overlay is receive-only, forever. It listens for slot-state, status-state, and appearance-changed; it never invokes commands. The settings window is the only invoke surface.
  2. Priority is color, never a chip. Three cues, same accent: the 3px edge, the stamp border/text, the slider's lit segments. No text codes.
  3. One easing, one register. cubic-bezier(0.22,1,0.36,1) everywhere; celebrations are one-shot and signal-keyed — never priority-keyed, never looping.
  4. Markdown is inline and escaped. code/bold/italic/line-breaks, React-node tokenizer, no dangerouslySetInnerHTML. Hook-supplied details are plain text, never markdown.
  5. Looping animations are compositor-only. transform and opacity only (plan 018). No background-*, box-shadow, filter, or layout animation on an infinite loop.
  6. Reduced motion ⇒ silence, not substitutes. Celebrations suppress entirely; the card, accent, stamp, and copy still render and announce.
  7. The settings preview mirrors the overlay CSS. src/settings/preview-overlay.css is a scoped copy of src/styles.css — change one, change both, in the same commit.
  8. Stamps are a lookup table, never parsing. Signal/priority/eventType drive the stamp via lib/presentation.ts; we never regex titles or bodies to decide what to show.

06Control panel

The Settings window — the app's only other UI surface — shares the overlay's near-black surfaces and signature easing but speaks a denser, form-first dialect: an 8-section sidebar nav, the rotation-order widget, the shortcuts cheatsheet's kbd styling, the form-input patterns (toggle, number field with unit suffix, priority toggle), and the footer button row. Mirrored from src/settings/settings.css and src/settings/SettingsApp.tsx as static markup — the overlay stays receive-only; this window is the only invoke surface.
