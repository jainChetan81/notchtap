notchtap — nudge UI demo

Static HTML mirror of src/App.tsx + src/styles.css — same classes, same keyframes, same shape/animation lookup as the real Tauri webview. Not wired to the real /notify endpoint or rust queue; this is a UI-only walkthrough of what each case looks like. TTL is sped up to 3s (real default is 8s) so you don't have to wait around.

1 — Presentation Mode

Decided at runtime via notchtap-detect reading NSScreen.safeAreaInsets.top — same compiled app, one runtime branch. In Notch mode, shape is looked up per event type (getMorphShape): grow for generic, pill for score_update/match_state. In HUD mode every event type renders as the same mini pill — there's no per-type distinction once you're off the notch.
429 — queue full, push rejected
⏸ Paused — Promotion disabled
↑ physical hardware notch · window renders flush against it
↑ plain menu bar, no cutout · window floats independently

2 — Push an event

generic/score_update/match_state are the three eventType values useVisibleNotifications.ts knows about. "Burst push" fires 6 pushes back-to-back to show Visible cap, Waiting queue, and the 429 rejection once Waiting is also full.

3 — Queue state

Visible: 0/3
Waiting: 0/5 (demo cap — real default is 50)
Paused: no
Paused: Promotion is disabled — new pushes are still accepted into Waiting (even if a Visible slot is free), and already-Visible items keep running their natural TTL and exit. Resume re-enables Promotion and drains Waiting into any free slots. Nothing is ever dropped by pausing.

Glossary reference (from CONTEXT.md)

Visible / Waiting
Visible = currently rendered, capped at max_concurrent (3). Waiting = accepted but not shown, FIFO, capped at max_queued (50 in prod).
Promotion
The engine moving a Waiting item into Visible. The frontend never decides this — it only animates whatever it's told is Visible.
TTL
How long an item stays Visible, measured from Promotion — not from when it was pushed. Waiting time never burns TTL.
Paused
Promotion disabled; pushes still buffer into Waiting; Visible items still exit on schedule.
