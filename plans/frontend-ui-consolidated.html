Engineering constraints to remember

No hover exists on the real window today — mechanism now chosen (plan 086, 2026-07-21). apply_overlay_native_config (src-tauri/src/lib.rs:544-580) sets set_ignore_cursor_events(true) unconditionally on the whole overlay (:554) and stays that way — the plan 086 spike found tauri-nspanel's already-pinned, already-vendored tracking-area support delivers real mouseEntered/mouseExited/mouseMoved on this exact window with zero change to that line (empirically verified: click-through stays intact while tracking still fires). See docs/design/hover-cursor-tracking.md. Every hover interaction in these prototypes is now unblocked in principle, but none of the four (081/082/084/idle expanded-on-hover) has actually wired the primitive yet — this is a chosen path, not a shipped feature.
Appearance preview mirror law. The Settings Appearance section renders the real StatusRailCard through src/settings/preview-overlay.css, a scoped mirror of src/styles.css. Any card change must update both files in the same commit (DESIGN.html Law #7).
063 idle-rail wrap ships before 079. Locked: flex-wrap on .src-rail, notch-mode width cap = cutout width, 270px floor, live-chip ellipsis, no visual restyle. Missing Steps/Test-plan sections; MacBook smoke-check owed.
