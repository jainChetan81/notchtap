## Why this matters

`src-tauri/icons/icon.png` (and every sibling — `icon.icns`, `icon.ico`,
the `Square*Logo.png` Windows tiles, `StoreLogo.png`) is still the literal
default artwork generated by `npm create tauri-app` — verified by reading
the file directly: teal and yellow interlocking circles, no notchtap
branding at all. This shows up in the dock, the tray, `Cmd+Tab`, and the
`.app`/`.dmg` bundle the operator installs. It's been this way since the
icons were first generated (16 Jul, before any of the v1-v6 feature work
landed) and nobody has touched it since.

## Current state

- `src-tauri/icons/` — 16 files, all Tauri-default, referenced from
  `src-tauri/tauri.conf.json:42-47` (`bundle.icon`) and implicitly by
  `tauri-cli`'s tray-icon default (`TrayIconBuilder`, `src-tauri/src/lib.rs:24,655`
  — check at execution time whether the tray icon needs its own smaller
  asset separate from the app icon, or reuses the same source).
- No design system precedent exists to draw from — `DESIGN.html`'s
  color system (teal `#55d6bd` / amber-ish priority accents, near-black
  surface) is the only established visual language in this repo and is a
  reasonable starting palette, but nothing icon-shaped has ever been
  designed.

## Decision needed (Step 0 — operator)

Before any artwork exists, resolve:

1. **Mark direction**: an abstract mark (e.g. referencing the notch/HUD
   card shape — the sharp-top/rounded-bottom silhouette already used by
   `.rail-card`), a wordmark/monogram (e.g. "nt"), or something else.
2. **Palette**: reuse `DESIGN.html`'s existing teal accent
   (`--accent: #55d6bd` at Medium priority) as the primary brand color, or
   pick something distinct from the in-app priority-accent system so the
   icon doesn't read as "this is a Medium-priority notification."
3. **Source of the artwork**: hand-drawn/vector (needs a design tool,
   likely outside this session's capability to produce well), or
   generated (AI image generation, then manually cleaned up to a flat
   vector-safe shape suitable for a 16px-1024px range — small sizes
   (16x16, 32x32 menu-bar/dock) need to stay legible, which busy/gradient-heavy
   generated art usually isn't).

## Next step

Once Step 0 is resolved, generate the full icon set (`tauri icon
<source.png>` regenerates every size/format this repo's `tauri.conf.json`
references from one 1024x1024 source — confirm the CLI is available via
`npx tauri icon --help` before assuming it). Replace every file in
`src-tauri/icons/` and rebuild. Verify: dock icon, tray icon, `Cmd+Tab`
switcher, and the `.dmg` volume icon all show the new mark, not the old
one — `npm run tauri dev` and a visual check covers dock/tray/switcher;
the `.dmg` icon only shows up in a full `npm run tauri build`.
